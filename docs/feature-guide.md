# 功能介绍书

## 产品名称

- 中文名：无限画布多模型聊天室
- 英文名：MeshCanvas

## 一句话介绍

一款把多模型对话、联网搜索、分支推演和管理运营能力放进同一张无限画布的本地 Web 产品。

## 产品能做什么

传统聊天工具通常一次只看一个模型、一条回答，信息很快被新消息覆盖，也不方便比较不同模型的差异。无限画布多模型聊天室把一次提问拆成多个并行回答，并以节点方式保留在画布上，让用户可以持续展开、继续讨论和沉淀结果。

它不是单纯的聊天窗口，而是一套多模型协作工作台。

## 核心功能介绍

### 1. 多模型并发回答

用户输入一条问题后，系统可以同时调用多个大模型。每个模型独立流式返回内容，用户能一边看生成过程，一边比较不同模型的回答风格、速度和质量。

适合场景：

- 对比多个模型答案
- 挑选更好的写作版本
- 交叉验证复杂问题
- 快速观察不同模型的思路差异

### 2. 无限画布展示

所有用户提问和模型回答都会以节点形式落在画布上，而不是被限制在一条线性会话里。用户可以拖动节点、缩放画布、平移视角，用更接近白板和脑图的方式组织 AI 输出。

适合场景：

- 做问题拆解
- 保存推理路径
- 按主题整理回答
- 长对话结构化沉淀

### 3. 分支对话

当某个模型的某一轮回答值得继续追问时，用户可以直接从该节点发起分支。这样既保留原始上下文，又能探索新方向，不会把多个思路混在一起。

适合场景：

- 从一个观点继续深挖
- 为不同假设建立独立路线
- 写作时同时保留多个版本

### 4. 多轮讨论

系统支持设置讨论轮数。开启后，多个模型不仅会回答用户问题，还会参考其他模型上一轮的内容继续讨论。这让产品从“多模型并发输出”升级成“多模型参与式讨论”。

适合场景：

- 辩论式内容生成
- 方案优缺点对比
- 复杂问题多角度分析
- 需要更深入推演的场景

### 5. 联网搜索

产品支持 Firecrawl 联网搜索，可以在模型回答前补充外部网页信息。用户可自行选择是否联网，也可以让系统先判断这个问题需不需要联网。

适合场景：

- 查询时效性信息
- 获取网页资料后再总结
- 减少模型纯凭记忆回答造成的偏差

### 6. 思考模式

开启“思考”后，模型会更偏向分析、验证和推理，而不是只给简短结论。这个模式适合要求更稳妥、更细致输出的任务。

适合场景：

- 复杂分析
- 研究问题
- 推理类写作
- 高风险信息校对

### 7. 圈选总结

用户可在画布上框选多个节点，让系统对选中内容做摘要。这可以把一组分散的模型回答快速压缩成可读结论。

适合场景：

- 汇总多模型结果
- 提炼长对话要点
- 为后续汇报准备摘要

### 8. 会话结题

当一次对话已经产生足够多的信息时，系统可以进一步生成结论或结题结果，帮助用户从“讨论过程”回到“最终输出”。

适合场景：

- 给研究过程收尾
- 输出最终判断
- 形成可执行结论

### 9. 多画布管理

用户可以按主题建立多张画布，例如“产品规划”“竞品分析”“写作草稿”“代码讨论”等，不同内容彼此隔离，避免在同一张画布中堆积过多节点。

适合场景：

- 多项目并行
- 按主题沉淀
- 长期使用时维持整洁

### 10. 用户账号与用量页

系统提供单独的账号页面，用户可以查看积分余额、今日消耗、近 7 天使用情况、历史总量和逐条用量明细。

适合场景：

- 自查使用成本
- 查看调用记录
- 理解不同模型的消耗差异

### 11. 自定义 API Key

用户可以按模型配置自己的 API Key，并选择是否继续使用平台积分计费。这使平台同时支持“统一运营”和“用户自带资源”两种使用方式。

适合场景：

- 自己已有模型额度
- 希望保留统一界面但不走平台计费
- 需要按人隔离资源

## 管理后台功能介绍

产品不只是前台对话工具，也是一套完整的后台管理系统。

### 1. 用户管理

管理员可以查看用户列表、充值、扣减积分、重置密码、调整角色。

### 2. 模型配置

管理员可以统一配置：

- 平台 API 地址
- 平台 API Key
- 接口格式
- 模型列表
- 预处理模型
- 用户自定义 Key 对应的独立网关

### 3. 联网搜索配置

管理员可以统一配置 Firecrawl Key、国家代码和超时时间，控制整个系统是否允许联网搜索。

### 4. 模型定价

管理员可按模型设置输入和输出 Token 的积分价格，为平台计费提供基础。

### 5. 用量统计

后台支持查看用户和模型维度的 Token 消耗、调用次数和积分支出。

### 6. 系统配置

管理员可设置新用户默认积分、低余额提醒阈值、搜索积分消耗和是否开放注册。

### 7. 审计日志

后台会记录关键管理行为，便于追踪平台运营操作。

## 产品特色总结

相比普通聊天产品，这个产品的特色在于：

- 同时看多个模型，而不是单模型串行切换
- 用画布保存思路结构，而不是线性聊天记录
- 支持从任意回答继续分支，而不是只能接在最后一条消息后面
- 支持讨论轮次和联网搜索，而不是单次静态回答
- 自带后台管理和积分运营机制，而不只是一个前端界面

## 总结

无限画布多模型聊天室是一款兼顾“多模型使用体验”和“平台运营能力”的产品。前台强调并发、可视化和分支推演，后台强调配置、定价、统计和审计，因此它既可以作为个人高级 AI 工作台，也可以作为团队内部的多模型平台基础版本。
